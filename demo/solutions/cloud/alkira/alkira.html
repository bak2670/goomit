멀티 클라우드 네트워킹 플랫폼

Cloud Networking as a Service(CNaaS)

멀티 클라우드로 업무 환경이 빠르게 진화되고 있으며
다양한 네트워킹 환경(홈 오피스, 지점, 본사, 코 로케이션)을 유연하게 연결하고,
연결된 요소 간의 논리적 구분 및 그룹화를 통하여 일관된 정책 적용 및
네트워크 서비스를 구현할 수 있는 멀티 클라우드 네트워킹 플랫폼입니다.

자료다운로드
Cloud Networking as a Service 특징
Multi Cloud 환경과 기업 내 Branch,
Home Office 및 Data Center 간 기업 전용
① Global Cloud Backbone Service를 표방하며
다양한 Branch 환경 (IPsec, SD-WAN, Remote Access VPN)을
Multi Cloud (AWS, Azure, GCP, OCI 등)와
상호 연결하는 기업 전용 Backbone Network망 구성 지원
② 해당 Branch 및 Cloud 자원의 위치 기준으로
인접 서비스 PoP(Regional Cloud Hub)을 선정하고
Branch 및 Cloud 연결을 Dashboard의 단순한 Click으로 몇 분 내 구성 가능
사설 Backbone을 이용한
일관된 정책 적용
① 구성된 사설망은 논리적(Macro-Segmentation)으로
Network 구분 및 자원에 대한 그룹화 (Micro-Segmentation)를 통하여
다양한 정책 적용 가능(Traffic, Routing, NAT 정책)
② Branch 및 Cloud 내 보안 정책을 개별 단위에서 하지 않고
Backbone 단위에서 중앙 집중적으로 적용 가능
③ 방화벽 서비스 등 고급 보안 적용이 필요시
Backbone내 방화벽 서비스를 활성화하여 방화벽으로 경유될 Traffic만
선별적으로 처리 가능
3rd Party Vendors 서비스
Alkira Marketplace
① 네트워킹, 네트워크 서비스의 복잡성과 연동 간 제약사항을
최소화하도록 다양한 Solution 연동 및 Cloud 간 API 통합
② Backbone 내 연동 가능 Solution: SD-WAN (Cisco, VMware, Aruba),
Firewall(Paloalto, Checkpoint, Fortinet, Akamai Prolexic, Cisco),
Cloud Security (Zscaler) 등
③ 기업 전용의 Private Backbone Network Cloud 관리를 위한
통합된 Dashboard 제공

이런 기업에게
CNaaS가 필요합니다.

  • 멀티 클라우드(ex.aws, azure, gcp 등)를 이용하거나, 멀티 리전을 이용 중인 기업
  • 일부 데이터 센터만 클라우드로 이전하여 하이브리드 형태로 인프라를 운영 중인 기업
  • 클라우드 데이터 센터와 On-Premise 데이터 센터 간의 전용선을 이용하거나 SSL VPN을 이용 중인 기업
